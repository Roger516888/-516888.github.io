GitHub Pages 知識樹狀圖

  • 1. 基本概念
    • 1.1 什麼是GitHub Pages？
      • 1.1.1 定義：GitHub提供的免費靜態網頁託管服務
      • 1.1.2 用途：展示個人網頁、項目DEMO（如地圖分佈）
      • 1.1.3 特點
        • 免費（公開倉庫）
        • 支援HTML/CSS/JS
        • 自動託管於username.github.io
    • 1.2 限制
      • 1.2.1 僅限靜態內容（無後端程式，如PHP）
      • 1.2.2 流量限制（每月100GB）
      • 1.2.3 檔案大小限制（每個檔案<100MB）
    • 1.3 與GitHub的關係
      • 1.3.1 基於GitHub倉庫
      • 1.3.2 公開倉庫免費，私有需付費
  • 2. 前置準備
    • 2.1 帳戶需求
      • 2.1.1 GitHub帳戶（註冊於github.com）
      • 2.1.2 驗證Email（啟用Pages功能）
    • 2.2 工具準備
      • 2.2.1 瀏覽器（Chrome、Firefox）
      • 2.2.2 文字編輯器（Notepad++、VS Code）
      • 2.2.3 可選：Git（命令列上傳）
    • 2.3 檔案準備
      • 2.3.1 HTML檔案（如leaflet_map.html）
      • 2.3.2 支援檔案（CSS、JS，若有）
      • 2.3.3 確認檔案可本地運行
  • 3. 操作流程
    • 3.1 創建倉庫
      • 3.1.1 命名規則：username.github.io
      • 3.1.2 設定公開（Public）
      • 3.1.3 初始化README（可選）
    • 3.2 上傳檔案
      • 3.2.1 方法1：網頁拖曳
        • 前往倉庫頁面
        • 拖曳leaflet_map.html
        • 提交（Commit）更改
      • 3.2.2 方法2：Git命令
        • 安裝Git
        • 複製倉庫（git clone）
        • 新增檔案（git add）
        • 提交（git commit）
        • 推送（git push）
    • 3.3 啟用GitHub Pages
      • 3.3.1 進入Settings > Pages
      • 3.3.2 選擇Branch（main）
      • 3.3.3 儲存，等待部署（1-2分鐘）
    • 3.4 訪問網頁
      • 3.4.1 網址：https://username.github.io/leaflet_map.html
      • 3.4.2 確認地圖顯示（25個標記，212台）
      • 3.4.3 分享網址
  • 4. 進階應用
    • 4.1 更新DEMO
      • 4.1.1 修改HTML（例如新增標記）
      • 4.1.2 重新上傳
      • 4.1.3 自動更新（5-10分鐘生效）
    • 4.2 自訂域名
      • 4.2.1 購買域名（例如yourdomain.com）
      • 4.2.2 設定DNS（指向GitHub IP）
      • 4.2.3 Pages設定中輸入域名
    • 4.3 隱私管理
      • 4.3.1 公開倉庫：所有人可見
      • 4.3.2 私有倉庫：需GitHub Pro
      • 4.3.3 密碼保護（自訂JS）
    • 4.4 整合其他工具
      • 4.4.1 與地圖DEMO結合（Leaflet/Plotly）
      • 4.4.2 添加心智圖（D3.js展示）
      • 4.4.3 分析流量（Google Analytics）
  • 5. 常見問題
    • 5.1 網頁不顯示
      • 5.1.1 檢查網址（是否包含.html）
      • 5.1.2 確認Pages已啟用
      • 5.1.3 等待部署（可能延遲）
    • 5.2 地圖載入失敗
      • 5.2.1 檢查網路（Leaflet需連線）
      • 5.2.2 驗證JS路徑（unpkg.com）
      • 5.2.3 本地測試HTML
    • 5.3 檔案上傳錯誤
      • 5.3.1 確認檔案大小（<100MB）
      • 5.3.2 檢查Git提交（無衝突）
      • 5.3.3 重新上傳
    • 5.4 其他問題
      • 5.4.1 流量超限（升級計劃）
      • 5.4.2 瀏覽器相容性（測試Chrome）
      • 5.4.3 聯繫支援（GitHub Help）
